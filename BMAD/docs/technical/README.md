# Technical Documentation Index

This directory contains technical documentation for the BMAD project.

## Documentation Categories

### API Documentation
- API specifications and endpoint documentation
- Authentication and authorization guides
- Request/response examples and schemas
- SDK and integration guides

### System Architecture
- High-level system architecture diagrams
- Component interaction maps
- Data flow documentation
- Infrastructure diagrams

### Database Documentation
- Schema definitions and relationships
- Migration guides and procedures
- Performance optimization notes
- Backup and recovery procedures

### Development Guides
- Setup and installation instructions
- Local development environment configuration
- Build and deployment procedures
- Troubleshooting guides

### Integration Documentation
- Third-party service integrations
- External API usage documentation
- Webhook configurations
- Service dependencies

## Document Templates

Use the following templates when creating new technical documentation:

- `api-endpoint-template.md` - For documenting individual API endpoints
- `architecture-decision-record-template.md` - For architectural decisions
- `integration-guide-template.md` - For integration documentation
- `troubleshooting-guide-template.md` - For troubleshooting procedures

## Documentation Standards

1. **Format:** Use Markdown for all documentation
2. **Naming:** Use kebab-case for file names
3. **Structure:** Follow the template structures provided
4. **Updates:** Keep documentation current with code changes
5. **Review:** All technical docs should be peer-reviewed

## Maintenance

- Review documentation quarterly for accuracy
- Update with each major release
- Archive outdated documentation appropriately
- Ensure examples and code snippets are tested
